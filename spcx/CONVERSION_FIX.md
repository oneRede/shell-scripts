# 数据换算问题修正说明

## 问题发现

2026-07-13，在首次测试时发现数据被放大了10倍：
- 显示价格: $1,453.00
- 实际价格: $145.30
- 误差倍数: 10倍

## 根本原因分析

### 东方财富 API 字段说明

东方财富网的API返回的价格字段有不同的换算规则：

1. **商品价格字段（如黄金）**
   - 字段: f58
   - 换算: **不需要除以任何数值**，直接使用
   - 示例: 黄金价格 API 返回 4067.99，实际就是 $4,067.99

2. **A股/人民币资产**
   - 换算: **除以 100**
   - 原因: 价格以"分"为单位存储（1元 = 100分）

3. **美股/美元资产**
   - 换算: **除以 1000**
   - 原因: 可能使用更小的单位存储（如毫元，1美元 = 1000毫）

### 实际测试数据

```
API 原始返回值（推测）:
f43 (最新价) = 145300
f46 (开盘价) = 150130
f44 (最高价) = 150570
f45 (最低价) = 145070

错误换算（除以100）:
最新价: 145300 / 100 = 1453.00 ❌
开盘价: 150130 / 100 = 1501.30 ❌

正确换算（除以1000）:
最新价: 145300 / 1000 = 145.30 ✅
开盘价: 150130 / 1000 = 150.13 ✅
```

## 验证方法

如果未来需要验证换算规则是否正确，可以：

1. **对比已知股票**
   - 查询 Tesla (TSLA) 等知名美股
   - 对比东方财富网页显示价格和API返回值

2. **检查涨跌幅**
   - API 返回的 f170 字段（涨跌幅）应该是百分比值
   - 如果涨跌幅正确（-4.51%），但价格错误，说明换算有问题

3. **多数据源对比**
   - 同时查询 Yahoo Finance、新浪财经等
   - 确认哪个价格是正确的

## 经验总结

1. **不同资产类型有不同的换算规则**
   - 不能假设所有字段都用相同的换算方式
   - 需要先测试验证再部署

2. **使用涨跌幅作为验证**
   - 涨跌幅通常是准确的
   - 可以反推正确的价格

3. **添加合理性检查**
   - SpaceX 股价不太可能超过 $500
   - 如果价格看起来异常，需要重新检查换算
