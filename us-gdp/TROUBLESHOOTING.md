## 问题现象
- **失业率**和**通货膨胀率**间歇性获取失败
- 日志显示：`失业率: 未获取`、`通货膨胀率: 未获取`
- 其他指标（GDP、PCE、消费者信心指数）正常

## 根本原因分析

### 1. 数据源空值问题
FRED数据源中存在空值行，例如：
```
2025-10-01,
```

### 2. 缺少健壮性检查
原始代码只检查 `if len(row) == 2 and row[1]:`，但在某些情况下：
- 空字符串可能通过检查
- 日期格式验证缺失
- 缺少长度验证导致索引错误

### 3. macOS兼容性问题
原修复尝试使用 `timeout` 命令，但macOS默认不包含此命令（需要通过 `brew install coreutils` 安装 `gtimeout`）。

## 修复方案

### 改进点
1. **增强空值过滤**：`if len(row) == 2 and row[1] and row[1].strip()`
2. **日期格式验证**：`if len(date) >= 7` 确保日期格式正确
3. **错误重定向**：`2>/dev/null || echo ""` 防止Python错误导致脚本中断
4. **网络超时控制**：使用 `curl --max-time 10` 替代 `timeout` 命令
5. **AWK空值检查**：在GDP函数中添加 `if (NF == 2 && $2 != "")` 验证

### 修改的函数
- `get_gdp_growth()` - 添加AWK空值检查和curl超时
- `get_unemployment()` - 增强空值过滤和日期验证
- `get_inflation()` - 增强空值过滤和日期验证
- `get_pce()` - 增强空值过滤和日期验证
- `get_consumer_confidence()` - 增强空值过滤和日期验证
